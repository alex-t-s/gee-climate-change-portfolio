# Portfolio Workflow

## End-to-End Geospatial Analysis Workflow

This portfolio follows a standardized geospatial workflow designed to ensure reproducibility, transparency, and consistency across all projects.

---

## Phase 1 — Problem Definition

Each project begins with a spatial question or environmental management challenge.

Examples include:

* Where are climate risks concentrated?
* Which locations are most suitable for solar development?
* How has wetland hydroperiod changed over time?
* Which populations are most exposed to environmental hazards?

---

## Phase 2 — Data Acquisition

Datasets are collected from authoritative sources through Google Earth Engine.

Examples:

* TerraClimate
* Landsat
* SRTM
* WorldPop
* JRC Global Surface Water

---

## Phase 3 — Data Preparation

Tasks include:

* Study area definition
* Spatial clipping
* Variable selection
* Raster normalization
* Environmental preprocessing

---

## Phase 4 — Geospatial Analysis

Analytical techniques include:

### Raster Analysis

* Reclassification
* Weighted overlays
* Raster algebra

### Remote Sensing

* Environmental trend detection
* Temporal analysis
* Surface-water monitoring

### Terrain Analysis

* Slope calculation
* Aspect calculation
* Elevation modeling

### Vulnerability Assessment

* Exposure mapping
* Risk indexing
* Environmental suitability scoring

---

## Phase 5 — Statistical Summaries

Project outputs are summarized into CSV tables.

Examples include:

* County summaries
* Parish summaries
* Statewide statistics
* Hydroperiod metrics
* Exposure rankings

---

## Phase 6 — Visualization and Cartography

Outputs are transformed into:

* Dashboard maps
* Hero maps
* Portfolio cover graphics
* GIS visualizations

Emphasis is placed on clarity, visual hierarchy, and communication of analytical findings.

---

## Phase 7 — Documentation and Version Control

All projects include:

* README documentation
* Methodology reports
* Exported outputs
* Source code
* Version control through GitHub

---

## Deliverables

Each project contains:

* Google Earth Engine source code
* Exported raster products
* Statistical tables
* Dashboard screenshots
* Hero maps
* Technical documentation

This workflow reflects the process used by GIS analysts, environmental consultants, planners, and geospatial professionals when developing decision-support products from spatial data.
